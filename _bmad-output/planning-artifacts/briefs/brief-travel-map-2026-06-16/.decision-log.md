## 2026-06-16

- Intent: **Create**. Coaching path chosen by user.
- Source material: `_bmad-output/brainstorming/brainstorming-session-2026-06-16.md` + saved concept memory.
- Stake read: **passion project / personal-tool-first.** Primary user is Simon himself; success = he returns to the app to re-live where he's been. Right-size accordingly (not investor-grade).
- Target user (Simon's words): people who want to document their trips/travels in one app and come back again and again to see those precious memories.
- **Central challenge named:** retention is the category's hardest problem (people log once, never return). Simon's own success metric IS returning — so the brief must stare at this directly.
- **Retention loop (core):** notification → "on this day" memory resurfacing → user taps → re-lives → may add forgotten memories. Notification-driven pull (not user-initiated).
- **Design principle:** notifications are *memory delivery*, never engagement-baiting nags. "Here's you in Kyoto 2 years ago" not "you haven't logged in a while."
- **New parked idea:** auto-stitch photos + notes into a short memory video/reel. Fast-follow, NOT MVP. (Polarsteps "trip reels" precedent.)
- **Trajectory clarified:** personal-first, PUBLIC second. Test on self, then open to others. v1 builds accounts/privacy/durability for real (not faked) because real users are coming. ("Make it public" = open product to users, NOT public/social map sharing — those are different; sharing stays deprioritized.)
- **Constraint:** web-first but mobile-ready. Tech stack must not trap on web. API-first backend + credible mobile path (PWA/responsive or native-extensible). Captured in addendum for architecture step.
- First 4 brief sections (Exec Summary, Problem, Who, Differentiators) drafted and approved by Simon (minor framing edits for public trajectory applied).
- **Scope decision (confirmed by Simon):** retention loop ("on this day" resurfacing + memory-delivery notifications + per-place memory view) is IN the v1 MVP, not deferred. Rationale: without it, v1 is just another scratch map; the loop is the product's reason to exist. Accepts the added build time.
