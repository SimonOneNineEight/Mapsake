## Visual identity — Color palette (LOCKED 2026-06-16)

Direction: **Sepia Parchment** — warm, muted, milky-canvas atlas. Chosen via iteration over color-themes-1 → 4 (artifacts in `.working/`). Final source of truth: `.working/color-themes-4.html`.

User's words that shaped it: "warm feeling", "muted", "milk not white", "canvas/paper texture". Visited regions are the hero; unvisited regions are the bare paper (matches PRD "unvisited stay plain"); only the soft border delineates them.

**Light (default):**
- canvas-bg `#F2E8D5` (milky parchment; also = unvisited land)
- surface `#FBF4E4` (panels/cards)
- region-visited-fill `#B5663E` (HERO terracotta clay — Mapsake's signature hue)
- region-unvisited `#F2E8D5` (= canvas-bg; unvisited land IS the paper)
- region-border `#C7B79A` (soft; sole delineator of regions)
- text-primary `#3A2E22`
- text-muted `#8A7456`
- accent `#C8893B` (glow for the "re-living" region, links, active states)

**Dark ("Lamplight"):**
- canvas-bg `#2C2621` (cozy warm charcoal-brown — not blue-cold, not pure black)
- surface `#37322B`
- region-visited-fill `#C4703F` (terracotta glow — brand hue carries across modes)
- region-unvisited `#383229`
- region-border `#4D463B`
- text-primary `#E8E2D8`
- text-muted `#978F84`
- accent `#E08A4E`

**Decided rules:**
- NO free user color-picker in v1. Rationale: visited state is binary (one fill color suffices); multiple colors would imply out-of-scope categories; terracotta is the brand face; contrast/accessibility risk; scope. Curated map-theme presets are a documented fast-follow, not v1.
- Dark mode: palette is designed. **DECIDED by Simon 2026-06-16: DEFER to phase 2.** v1 ships LIGHT-ONLY; the Lamplight dark palette stays fully specified in DESIGN.md and is ready to enable when wanted. Avoids dark-theme design+test scope on every screen for v1.

## Visual identity — Typography (LOCKED 2026-06-16)

Direction: **Editorial Mix** — warm serif for emotional/display moments, humanist sans for working UI + body. Chosen via type-specimens-1 (5 directions, macOS stand-ins) → type-specimens-2 (3 real-web-font pairings). Final artifact: `.working/type-specimens-2.html`.

- **Display / serif: Newsreader** (Google Fonts, variable). Used for: wordmark "Mapsake", place/memory titles ("Kyoto"), large emotional headings, the tagline in italic. Weights ~400/500/600 + italic 400.
- **Body + UI / sans: Nunito Sans** (Google Fonts, variable). Used for: notes, dates, buttons, labels, settings, dense screens. Weights ~400/600/700.
- **Map labels:** Nunito Sans, UPPERCASE, letterspaced (~0.12em), small (~12–13px), muted color. Quiet cartographic touch without going engraved.
- **Wordmark "Mapsake":** set in Newsreader for now (keepsake voice). A bespoke logotype is a possible later refinement, not v1-blocking.

Indicative type scale (from specimens, refine in DESIGN.md at Finalize):
- Wordmark ~48px Newsreader 500/600
- Place/memory title ~28–30px Newsreader
- Card/section heading ~20px Newsreader
- Body note ~15–16px Nunito Sans 400
- UI / button / date ~13–14px Nunito Sans 500/600
- Map label ~12–13px Nunito Sans 600 tracked caps

Rejected alternates (kept for rationale): Lora (softer serif — less editorial character than Newsreader); Inter (cooler sans — less warm than Nunito Sans).

## Map + Memory layout model (LOCKED 2026-06-16)

Wireframes: `.working/layout-map-memory-1.html`. Honors PRD hard rule: re-live landing = map + memory together, never slideshow/takeover-first.

**Wide screens (≥ ~1024px): side-by-side split.**
- Map ~60–65% (left), memory panel docked **right** ~35–40%.
- Map stays interactive; tapping another region swaps the panel content. Close (×) on panel.

**Phone (portrait): ONE draggable bottom sheet with THREE snap points.**
- **Default landing = half-height** (~lower 45–55%): map glowing above, memory below. Every re-live LANDS here → "together" is the primary path. Drag handle at sheet top.
- **Expanded** (drag up, ~85%): photos dominate, a sliver of glowing map remains visible.
- **Full** (drag up further): memory fills screen for photo immersion, with pull-down / "▾ back to map" to return.
- **Principle: full-screen is user-CHOSEN via drag, never the landing state.** This merges the former "takeover" alternate into the top snap of the same sheet rather than a separate screen.

Decided by Simon: default height = half (as drawn, frame 2); desktop panel docked right.

## Notification / re-live trigger model (LOCKED 2026-06-16)

Resolves PRD's explicit OPEN(UX) item on cadence. Reference studied: Apple Photos "Memories / On This Day" — curation over completeness, sparse frequency, sensitivity controls, clean off switch.

- **Curated, max ONE per day.** Hard daily ceiling. Fire only on a genuine "on this day" anniversary; most days are silent. Skip thin/low-value entries so every buzz is worth it.
- **No engagement nags ever.** No "you haven't opened Mapsake" / invented occasions. Notification = a real memory resurfacing.
- **Multiple matches same date → pick one, hint at the rest.** Notification names one place; app surfaces "N more from this day" on landing.
- **Tie-breaker [DEFAULT, revisitable]:** oldest memory wins (more years-ago = more emotional), photos as secondary tiebreaker so it leads somewhere visual.
- **Default delivery time [DEFAULT, revisitable]:** evening (matches the quiet couch re-live mood from the brief). User can change in settings.
- **Per-memory / per-place MUTE** (stolen from Apple): "feature this place less / mute this memory" — kindness for places tied to an ex or a hard time. Reinforces "for yourself, on your terms."
- **Global off switch** in settings, no dark patterns.
- Visual-first reward: notification deep-links to map + memory together (photos present) — already handled by the layout model.

Simon approved this plan 2026-06-16. The two [DEFAULT] picks (oldest tie-breaker, evening time) were not explicitly chosen by him — adopt as defaults, easy to change.

## Empty / quiet states (LOCKED 2026-06-16)

Mock: `.working/empty-states-1.html`. Principle: **absence is normal, never a failure.** No sad-faces, no "0 photos", no empty/broken slots, no progress meters, no "incomplete" badges, no "complete your profile" nags.

- **Blank map (new user):** all parchment, no terracotta; one soft centered line "Tap anywhere you've been to begin" + faint serif-italic sub-line. Airy, an invitation not a void. Fades once first region is colored.
- **Bare visited mark (most common state):** memory panel shows place name + calm optional "+ Add photos / + Write a note". Looks complete as-is. No date shown when absent (no "Date: —").
- **Partial memory:** show what's present (e.g. the note); missing pieces appear only as quiet optional offers ("+ Add photos", low-emphasis "+ add date"), never errors or blank slots.

## Onboarding / depth bet (LOCKED 2026-06-16)

**Bet: users mark BREADTH-first in onboarding, add memory DEPTH later (mostly via the re-live loop). Optimize onboarding for fast marking; never push memory entry.** Rationale: one-tap mark vs high-effort recall+photos+writing; the filled map is the onboarding payoff; backfilling years-old memories is hardest at signup; matches brief thesis "remember, re-live, occasionally add" + "no pressure to document everything at once"; analogous apps (mark-watched vs review) skew heavily to the light action. This makes the bare-mark-complete empty-state design the CORRECT default, not a compromise.

Onboarding shape: default-view question → rapid tap-to-mark (marks only, NO inline memory prompts) → at the end, ONE gentle skippable line ("Tap any place to add photos and notes whenever you like") → drop into the freshly colored map.

Counter-case held: **fresh-trip path (UJ-3)** is the opposite — recent trip, hot camera roll, user WILL want eager batch upload. Depth-entry must be frictionless + prominent THERE, just not woven into backfill. Two moments, two emphases.

Caveat (Simon's own point): Simon is first user → validate the bet directly. If he wants to add memories mid-backfill, revisit. Default: build for breadth.

### Typography — CJK addendum (LOCKED 2026-06-16)
The locked Latin pairing (Newsreader + Nunito Sans) is Latin-only. zh-TW companion preserving the serif-display / sans-UI identity:
- **Chinese display/serif: Noto Serif TC** (Song/Ming 明體 style — literary, keepsake warmth).
- **Chinese body+UI/sans: Noto Sans TC**.
- **Mixed-script rule:** font stacks list the Latin font FIRST, Noto TC AFTER — Latin glyphs (incl. "Mapsake" wordmark, English place names) render in Newsreader/Nunito Sans; CJK falls through to Noto TC.
- Demo: `.working/type-zhTW-1.html` (zh-TW + English side by side for cross-language brand coherence). **Fonts CONFIRMED by Simon 2026-06-16** (both Latin pairing and Noto Serif/Sans TC).
- **CJK title weight rule (LOCKED 2026-06-16):** memory/place titles (e.g. 京都) = **Noto Serif TC weight 500 (medium)**, NOT bold. Ming/明體 serif gets stern/heavy at 700; hierarchy carried by size + serif elegance. Bold 700 reserved for rare loud moments. Emphasis-via-terracotta-color (#B5663E) available as an on-brand option for special titles. Comparison: `.working/title-weight-zhTW-1.html`.

## Voice & Tone (LOCKED 2026-06-16)

Principles now; final microcopy drafted during build with a NATIVE-FLUENT zh-TW pass before launch (demo strings like 新增照片 / 稍後再補充 / 兩年前的今天：京都 are illustrative drafts, not final).
- **Quiet and warm, never a salesperson.** No hype, no exclamation-mark urgency, no FOMO.
- **Specific over generic.** "兩年前的今天：京都" beats "你有一則新回憶" — the specificity is the warmth.
- **Never nags, never scolds.** No "complete your profile", no "0 photos", no streak guilt. Absence is normal (matches empty-state rules).
- **Invites, never commands.** "稍後再補充" not "請填寫"; user controls pace.
- **Calm/literary register in Chinese**, matching the Song-serif keepsake feel — not a peppy app tone.

## Accessibility floor (consumer stakes) — 2026-06-16
- **Visited/unvisited is color-only → add a redundant non-color cue** (the one Mapsake-specific a11y risk). Options rendered in `.working/a11y-visited-cue-1.html` with deuteranopia simulation: A=texture/hatch, B=dot marker, C=heavier border. Facilitator recommends A (texture). **DECIDED by Simon 2026-06-16: Option A — terracotta + subtle texture/hatch, always-on.** Other-app context: texture = cartographic standard; marker/badge = app-collection convention (Letterboxd/Untappd) but gamified; border weight rare/muddy. GitHub contribution-graph color-only = cautionary tale.
- Text/list alternative to the map ("Places visited") for screen readers (doubles as a browse view).
- Honor `prefers-reduced-motion` (fly-to/glow → gentle fade fallback).
- Adequate tap targets for small regions (zoom / tap-assist); text scaling; visible focus states; check contrast on text + the soft `#C7B79A` borders.
- Scope: modest floor appropriate to a private personal keepsake; not over-built.
